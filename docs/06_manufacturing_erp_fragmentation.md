Manufacturing ERP Fragmentation Analysis
1. Summary

This document analyzes the manufacturing ERP portion of the onboarding case.

The production-related materials included Item, BOM, BOM Revision, Components, Routing, Work Center, Work Order, Assembly Build, Inventory Adjustment, MRP, Lot/Serial, BIN, Outsourced Manufacturing, Phantom settings, costing, GL impact, production reports, and raw material missing lists.

This was one of the most complex parts of the onboarding experience.

2. Manufacturing Is Not One Topic

Manufacturing ERP is not a single module from the learner’s perspective.

It combines:

master data
inventory
production planning
work execution
costing
accounting
purchasing
vendor management
location management
intercompany processes
reporting

When these layers are introduced together, the learner faces domain overload.

3. Master Data vs Transaction

A major difficulty was the mixing of master data and transaction data.

Master data includes:

Item
Assembly Item
Serialized Assembly
BOM
BOM Revision
Component
Routing
Work Center
Costing Method
Unit Type
Location
Planning Category

Transactions include:

Purchase Order
Inventory Adjustment
Work Order
Work Order Build
Assembly Build
Inventory Transfer
Item Receipt
MRP run
Production report

A new employee must first understand the difference between setup information and business events.

If both appear together without explanation, the manufacturing ERP structure becomes hard to understand.

4. BOM Complexity

BOM appeared in multiple forms.

It appeared as a real customer BOM, an ERP upload template, a NetSuite record, a BOM Revision, a component list, a CSV import sequence, and a production-costing reference.

This creates conceptual ambiguity.

The learner may not know whether “BOM” means:

engineering BOM
manufacturing BOM
NetSuite BOM record
BOM Revision
component upload file
cost structure
production execution structure

Without a glossary and hierarchy, this creates onboarding confusion.

5. Outsourced Manufacturing Complexity

Outsourced manufacturing adds another layer of complexity.

It may involve:

vendor manufacturing location
outsourcing charge item
outsourced purchase order
work order generation
receipt process
component consumption
backflush
vendor cost
inventory ownership
GL impact

This topic should not be introduced as a simple extension of BOM unless the learner already understands basic manufacturing flow.

6. Video-Based Compression

Some production materials were video-based screen-sharing sessions.

These videos included live system screens, training slides, Word documents, Excel templates, and NetSuite sandbox demonstrations.

Video can help, but it also creates a review problem.

Without timestamps, module headings, practice files, and checklists, a long video becomes difficult to search and reuse.

A screen-sharing session is not the same as a structured training module.

7. Data Quality Burden

Production materials also included missing raw material lists, upload templates, costing logic, and production report forms.

These are not beginner exercises.

They are data quality and implementation artifacts.

This means the employee was exposed not only to manufacturing concepts, but also to the consequences of incomplete master data.

8. CWI Interpretation

From a CWI perspective, manufacturing ERP fragmentation is a master-data and workflow-alignment problem.

The case shows how Item, BOM, Routing, Work Order, MRP, costing, and inventory movement must be aligned before ERP implementation can succeed.

It also shows how onboarding can fail when a new employee is exposed to these artifacts before the conceptual layers are separated.

9. Key Finding

The manufacturing materials were not simply difficult. They were structurally compressed.

Training materials, customer BOMs, ERP templates, system screens, transaction examples, costing logic, and data-quality issues were mixed together.

This created a manufacturing ERP onboarding overload.
